# XBOW Evaluation Workflow

This runbook defines a repeatable method to evaluate Vantix against XBOW-style targets with timed black-box and white-box runs.

Use this only for authorized lab targets.

## Goals

- Compare black-box and white-box performance on the same target.
- Capture timing, vectors, findings, browser coverage, and ground-truth match.
- Produce publishable summary artifacts for success analysis.

## Prerequisites

- Vantix API/UI running.
- An operator account available for API login.
- XBOW target reachable from the Vantix host.
- Optional source path/repo available for white-box runs.

Known devwork node from your shared memory context:

- Host: `192.168.1.99`
- SSH: `ssh -i ~/.ssh/dashburg_remoteops -o IdentitiesOnly=yes trilobyte@192.168.1.99`

## Files

- Runner: `scripts/xbow-eval-run.py`
- Aggregator: `scripts/xbow-eval-aggregate.py`
- Manifest template: `docs/examples/xbow-eval-manifest.example.json`

## 1) Prepare the Manifest

Copy and edit the template:

```bash
cp docs/examples/xbow-eval-manifest.example.json artifacts/xbow-eval/manifest.json
```

For each case, set:

- `target` (URL/IP with correct port)
- `black_box.objective`
- `white_box.source_input` (if white-box is used)
- `success_thresholds`
- `ground_truth` needles

Use strict targets (`http://host:port`) so browser assessment does not miss non-default web ports.

## 2) Run Timed Evaluation

```bash
python3 scripts/xbow-eval-run.py \
  --manifest artifacts/xbow-eval/manifest.json \
  --api-base http://127.0.0.1:8787 \
  --username admin \
  --password 'REPLACE_ME' \
  --mode pentest
```

Useful options:

- `--case-id <id>` run one case
- `--skip-white-box` / `--skip-black-box`
- `--timeout-seconds 2400`
- `--output-dir <path>`

Output:

- Per-case JSON files
- `summary.json`
- `summary.md`
- Run report package artifacts generated by Vantix:
  - `run_report.md`
  - `run_report.json`
  - `comprehensive_security_assessment_report.md`
  - `comprehensive_security_assessment_report.json`
  - `artifact_index.json` (+ `artifact_index.md`)
  - `timeline.csv`

Default output root:

- `${SECOPS_RUNTIME_ROOT}/reports/xbow-eval/<run-id>/`

## 3) Aggregate Across Multiple Suites

```bash
python3 scripts/xbow-eval-aggregate.py \
  --input ${SECOPS_RUNTIME_ROOT}/reports/xbow-eval \
  --output-dir artifacts/xbow-eval/aggregate
```

Outputs:

- `aggregate.json`
- `rows.csv`
- `aggregate.md`

The aggregate report computes:

- black-box and white-box success rate
- average elapsed time
- average vectors/findings/browser routes
- average ground-truth coverage
- white-box minus black-box deltas

## 4) Suggested Publish Format

When posting XBOW success analysis, include:

1. Target + authorization scope
2. Case count and run window
3. Black-box metrics
4. White-box metrics
5. White-box deltas vs black-box
6. Top validated findings with evidence links
7. Method constraints (noise/policy approvals)

## Method Notes

- Keep black-box and white-box objectives explicit and separate.
- Do not leak credentials/secrets in artifacts.
- Keep proof and hypothesis distinct in final writeups.
- If UI is unstable, this workflow is API-driven and still produces complete evaluation outputs.
